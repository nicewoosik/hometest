# 404 에러 해결 가이드

## 🔍 문제

크롤링 중 404 에러가 많이 발생하고 다운로드가 실패합니다.

## ✅ 개선 사항

크롤링 스크립트를 개선했습니다:
1. URL 정규화 및 중복 슬래시 제거
2. 404 에러를 더 명확하게 표시
3. 대체 경로 시도 로직 추가
4. URL 매칭 개선

## 🚀 다시 크롤링하기

터미널에서:

```bash
cd /Users/tommyjang/ecstel-clone

# 기존 dist 폴더 삭제 (선택사항)
rm -rf dist

# 개선된 크롤링 실행
npm run crawl
```

## 📋 개선된 기능

- URL 경로 정규화
- 404 에러 명확한 표시
- 대체 경로 자동 시도
- 더 나은 에러 처리

## ⚠️ 참고

일부 파일은 원본 사이트에서 접근이 제한되거나 존재하지 않을 수 있습니다:
- 외부 CDN 리소스 (예: jQuery UI)
- 동적으로 생성되는 리소스
- 보호된 파일

이런 경우는 정상입니다. 가능한 파일만 다운로드됩니다.

## 💡 팁

크롤링 후에도 일부 리소스가 없으면:
1. 브라우저 개발자 도구(F12)에서 네트워크 탭 확인
2. 404 에러가 나는 파일 확인
3. 해당 파일들을 수동으로 확인하거나 제거
